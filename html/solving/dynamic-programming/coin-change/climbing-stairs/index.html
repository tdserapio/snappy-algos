Climbing Staircase


Authored by: Troy Serapio â€¢ Published: March 18, 2024



Interpreting the Problem Statement


The problem asks us to determine the number of distinct ways to climb a staircase with n steps. You can take either 1 or 2 steps at a time.


For instance, if there are 2 steps (n = 2), there are 2 ways to climb:

  • Take 1 step at a time (1 step + 1 step)

  • Take 2 steps at once


If there are 3 steps (n = 3), there are 3 ways to climb:

  • Take 1 step at a time (1 step + 1 step + 1 step)

  • Take 2 steps followed by 1 step (2 steps + 1 step)

  • Take 1 step followed by 2 steps (1 step + 2 steps)


Dynamic Programming Solution


Dynamic programming is a powerful technique for solving problems that can be broken down into overlapping subproblems. In this case, the number of ways to climb n steps can be determined by considering the number of ways to climb the preceding 1 and 2 steps. Here's a step-by-step breakdown of the dynamic programming approach:


Define the base cases:


Since DP keeps going to the previous states, the base cases are essentially like the "stop, this is the default answer." In this case, it would be:


  • There is 1 way to climb 0 steps (the starting position).

  • There are 1 way to climb 1 step.


Note that there is an edge case of "there is 0 ways to climb negative steps" if doing this in the top-down approach.


Create a DP table:


Initialize a DP table dp of size n + 1 to store the number of ways to climb i steps (where i ranges from 0 to n).


Then, after initializing the DP table, we fill it up from the bottom to the top:

  • For each step i (from 2 to n):

    • The number of ways to climb i steps is the sum of the ways to climb i - 1 steps (taking 1 step) and i - 2 steps (taking 2 steps).

    • Mathematically, dp[i] = dp[i - 1] + dp[i - 2].


The final answer is stored in dp[n], representing the number of ways to climb n steps.


Implementation

Here's a Python implementation of the dynamic programming solution (bottom-up):



    def climbStairs(n):
        if n == 0: # Base case
            return 1
        elif n == 1: # Base case
            return 1
        else:
            dp = [0] * (n + 1) # Make DP array
            dp[0] = 1 # Base case
            dp[1] = 1 # Base case
            for i in range(2, n + 1): # DP solution itself
                dp[i] = dp[i - 1] + dp[i - 2] # The mathematical version of the DP solution
        return dp[n] # return dp[n]

What's really interesting about this solution is that, instead of being exponential growth in runtime (if it were done simply using recursion), by caching our results and just referencing it later on, the runtime is only actually the size of the array, which is O(n) where n is the number of steps to climb.
